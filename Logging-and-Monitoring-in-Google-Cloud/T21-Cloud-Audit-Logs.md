# üìù Audit Logs in Google Cloud

Cloud Audit Logs provide visibility into **who did what, where, and when** in your Google Cloud environment. They are crucial for compliance, troubleshooting, and security monitoring.

---

## üìÇ Types of Audit Logs

Google Cloud maintains **four audit logs** for each project, folder, and organization:

1. **Admin Activity Audit Logs** ‚úÖ
   - Log entries for API calls or administrative actions that **modify configuration or metadata** of resources.
   - Examples: creating VM instances, changing IAM permissions.
   - Always **enabled**, retained for **400 days**, and available at **no charge**.
   - Viewing requires IAM role: `Logging/Logs Viewer` or `Project/Viewer`.

2. **System Event Audit Logs** ‚öôÔ∏è
   - Log entries for **Google Cloud system actions** that modify resource configurations.
   - Not driven by user actions; generated automatically by Google systems.
   - Always **enabled**, free, and retained for **400 days**.
   - Viewing requires IAM role: `Logging/Logs Viewer` or `Project/Viewer`.

3. **Data Access Audit Logs** üîê
   - Log API calls that **read configuration or metadata** of resources.
   - Includes **user-driven API calls** that create, modify, or read resource data.
   - **Excludes** publicly shared resources or resources accessible without logging in.
   - **Disabled by default** (except for BigQuery); default retention: **30 days**.
   - Viewing requires IAM roles: `Logging/Private Logs Viewer` or `Project/Owner`.

4. **Policy Denied Audit Logs** üõë
   - Records when **access is denied** to a user or service account by Google Cloud service.
   - Generated by default; storage **charges apply**.
   - Cannot be disabled, but **exclusion filters** can prevent ingestion into Cloud Logging.
   - Useful for monitoring policy violations.

## üîç Viewing and Filtering Audit Logs

- Navigate to **Logs Explorer** in Google Cloud Console.
- Filter by **Log name** (e.g., type `cloudaudit` for quick filtering).
- If a log type has **no entries**, it may simply be inactive.
- Use the **Query Builder** to create and refine queries.
- Check the [Cloud Audit Logs documentation](https://cloud.google.com/logging/docs/audit) for detailed guidance.

---

## üåê Access Transparency Logs

- Provide logs of **human Googler accesses** to your data (not automated systems).
- Available to enterprises with the appropriate **support packages**.
- Delivered in **near real-time** via:
  - APIs
  - Cloud Logging
  - Security Command Center
- **Difference from Cloud Audit Logs**:
  - Cloud Audit Logs: actions by members of your organization.
  - Access Transparency Logs: actions by Google personnel.

## üìå Summary

Audit logs and Access Transparency logs are essential for:
- Compliance and regulatory requirements
- Troubleshooting and root cause analysis
- Security monitoring and auditing
