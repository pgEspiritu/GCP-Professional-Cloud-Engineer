## ðŸ–¥ï¸ Platform as a Service (PaaS)
PaaS offerings **bind code to libraries** that provide access to the underlying infrastructure needed by applications.  

- This allows developers to focus more on **application logic** rather than infrastructure management.  
- Example: **App Engine** (Google Cloud PaaS service)  
- **Billing model:** Customers pay for the resources they **actually use**.

As cloud computing has evolved, the momentum has shifted toward **managed infrastructure and managed services**.  

Benefits:  
- Companies can concentrate on **business goals** instead of maintaining technical infrastructure.  
- Products and services can be delivered **faster and more reliably**.

## ðŸ› ï¸ Serverless Computing
Serverless computing is **another step in cloud evolution**.  

- Allows developers to focus on **code** rather than server configuration.  
- Eliminates the need for **infrastructure management**.  

Examples from Google Cloud:  
- **Cloud Run:** Deploy containerized microservices in a **fully-managed environment**.  
- **Cloud Run Functions:** Manage **event-driven code** as a **pay-as-you-go service**.

SaaS provides the **entire application stack** as a cloud-based service.  

- Applications run in the cloud and are **consumed directly over the internet** by end users.  
- No installation is required on local computers.  

Examples from Google Workspace:  
- **Gmail**  
- **Docs**  
- **Drive**
